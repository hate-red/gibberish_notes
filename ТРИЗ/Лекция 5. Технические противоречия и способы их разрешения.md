>Формула:
	Если улучшаем параметр A, то ухудшается параметр B

##### Технические противоречия в программной инженерии

|                      | VS  |                                       |
| -------------------- | --- | ------------------------------------- |
| Производительность   |     | Безопасность                          |
| Скорость разработки  |     | Качество кода                         |
| Гибкость архитектуры |     | Оптимизация под <br>конкретные задачи |
| Универсальность      |     | Специализация                         |
##### Особенности
- Противоречия часто возникают между нефизическими параметрами
- Решения могут быть реализованы без материальных ограничений
- Возможно создание множества копий решения без дополнительных затрат
- Изменения могут быть внесены на любом этапе жизненного цикла
##### Значение
- Можно четко определить суть проблемы
- Перейти от расплывчатой задачи к конкретной формулировке
- Дает возможность применить ТРИЗ

##### Матрица противоречий

Это таблица 39 на 39, где:
- По вертикали 39 параметров, которые можно улучшить
- По горизонтали - 39 параметров которые ухудшатся
- На пересечении способы разрешения противоречий

##### Классификация 39 параметров технических систем:
1. Физические и технические
2. Производительность и эффективность
3. Эксплуатационные параметры

##### Наиболее часто используемые приемы
1. Дробление (разделить объект на независимые части)
2. Вынесение (отделить от объекта мешающую часть)
3. Объединение (соединить однородные объекты)
4. Предварительное действие (заранее выполнить требуемое действие)

##### Алгоритм формализации
1. Определение проблемы
2. Выявление административного противоречия
3. Выявление технического противоречия
4. Выявление физического противоречия
5. Сопоставление а параметрами матрицы

##### Рекомендации
1. Использование конкретных метрик
2. Разделение системных и пользовательских противоречий
3. Учет контекста использования системы
4. Выделение ключевых параметров противоречия
