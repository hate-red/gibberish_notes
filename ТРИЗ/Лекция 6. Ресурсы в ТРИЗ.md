==*Ресурсы в ТРИЗ*== - все, что может быть использовано для решения задачи и улучшения системы без существенных затрат 

##### Особенности
1. Уже существуют
2. Часто остаются незамеченными или недоиспользованными
3. Могут быть преобразованы из одного вида в другой
4. Являются основой для идеальных решений

##### Значение ресурсов в инженерных системах
1. Основа для решения противоречий
2. Путь в ИКР
3. Источник инноваций
4. Основа для устойчивого развития

##### Общая классификация ресурсов
- **Вещественные** - всевозможные материальные объекты 
  *Аппаратное обеспечение, физические носители, инфраструктура дата центров*
- **Энергетические** - различные виды энергии 
  *Энергопотребление серверов и ЦОД, тепловыделение, энергия мобильных и встраиваемых ресурсов*
- **Пространственные** - физическое либо виртуальное пространство системы 
  *Дисковое пространство, оперативная память, UI*
- **Временные** - временные интервалы в работе системы 
  *Процессорное время, время разработки, время отклика*
- **Информационные** - данные и знания 
  *Пользовательские данные, логи, исходные код, документация*
- **Функциональные** - возможности выполнения полезных функций, которые уже есть в системе или могут быть реализованы с минимальными затратами 
  *API, расширяемость системы через плагины, переиспользование компонентов, полиморфизм и наследование*

## Понятия скрытых ресурсов
==*Скрытые ресурсы*== - ресурсы, которые уже есть в системе, но либо не воспринимаются как полезные или не используются в полной мере

##### Причины невидимости ресурсов
1. Недостаток знаний
2. Узкая специализация
3. Психологическая инерция
4. Традиционные подходы

##### Примеры
1. Тепловыделение процессоров как источник обогрева
2. Шум как источник случайных чисел
3. Ошибки пользователей как данные для улучшения UX
4. Неиспользуемые вычислительные мощности

1. Системные операторы
2. Метод маленьких человечков
